# 🔐 Kubernetes & EKS Security Learning Path

A structured roadmap to learn **Kubernetes** and **AWS EKS security** — from fundamentals to advanced real-world practices.

---

## 🧭 Phase 1: Kubernetes Security Fundamentals

Start here. These are the **core building blocks**.

### 1. Authentication vs Authorization

* **Authentication** → Who are you?
* **Authorization** → What can you do?

**Topics:**

* API Server basics
* `kubeconfig`
* Certificates & tokens

---

### 2. ServiceAccounts

* Identity for Pods
* Token mounting inside Pods
* Default ServiceAccount risks

---

### 3. RBAC (Role-Based Access Control) 🔥

> ⭐ Most important topic

**Learn:**

* Role vs ClusterRole
* RoleBinding vs ClusterRoleBinding
* Verbs: `get`, `list`, `watch`, `create`, etc.

**Practice:**

* Grant read-only access to Pods
* Restrict namespace-level access

---

### 4. Secrets & Config Security

* Kubernetes Secrets (**base64 ≠ encryption**)
* Encryption at rest
* RBAC-controlled access to secrets

---

### 5. Pod Security

* `securityContext`
* `runAsUser`, `fsGroup`
* Privileged containers

**Also learn:**

* Pod Security Standards (Baseline / Restricted)

---

### 6. Network Policies

* Control pod-to-pod communication
* Default allow vs deny model

---

## 🧭 Phase 2: Advanced Kubernetes Security

Deepen your understanding.

### 7. Admission Controllers

Control what gets created in the cluster.

**Examples:**

* Pod Security Admission
* OPA Gatekeeper
* Kyverno

---

### 8. Image Security

* Image scanning
* Trusted registries
* Avoid `latest` tag

---

### 9. Audit Logging

* Kubernetes audit logs
* Track: *who did what*

---

## 🧭 Phase 3: EKS-Specific Security (AWS Layer)

Map Kubernetes concepts to AWS.

### 10. IAM Basics (Prerequisite)

* IAM users, roles, policies
* AssumeRole
* Least privilege

---

### 11. IAM Roles for Service Accounts (IRSA) 🔥

> ⭐ Critical EKS concept

**Maps:**

```
Kubernetes ServiceAccount → AWS IAM Role
```

**Use cases:**

* Access S3, DynamoDB, etc.

---

### 12. EKS Authentication (`aws-auth` ConfigMap)

* Map IAM users/roles → Kubernetes RBAC
* Control cluster access

---

### 13. Node vs Pod Permissions

* Node IAM Role (EC2)
* Pod IAM Role (IRSA)

⚠️ **Avoid giving excessive permissions to node roles**

---

### 14. EKS Networking Security

* Security Groups
* VPC design
* Public vs Private clusters

---

### 15. Secrets Management in EKS

* AWS Secrets Manager
* External Secrets Operator

---

### 16. Logging & Monitoring

* CloudWatch Logs
* AWS GuardDuty
* AWS Config

---

## 🧭 Phase 4: Real-World Hardening

Think like a security engineer.

### 17. Best Practices

* Apply **least privilege** everywhere
* Avoid default ServiceAccount
* Use IRSA instead of node roles
* Restrict network traffic

---

### 18. Security Tools

* `kube-bench` → CIS benchmarks
* `kube-hunter` → penetration testing
* `trivy` → image scanning
* `falco` → runtime security

---

## 🧠 Recommended Learning Order (Quick View)

1. ServiceAccounts
2. RBAC
3. Secrets
4. Pod Security
5. Network Policies
6. Admission Controllers
7. IAM Basics
8. IRSA (EKS)
9. `aws-auth` ConfigMap
10. EKS Networking & Logging

---

## ⚡ Pro Tip

❌ Don’t learn **IRSA first**

✅ First understand:

* ServiceAccounts
* RBAC

👉 Then IRSA will make sense easily.
